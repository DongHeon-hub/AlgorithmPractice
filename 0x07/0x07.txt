덱
: 양쪽 끝에서 삽입과 삭제가 전부 가능한 자료구조-> 스택, 큐는 택의 특수한 경우라고 할수있음
자료구조에서의 덱은 deque, double ended queue라는 뜻을 가지고 있음

덱의 성질
1. 원소의 추가가 O(1)
2. 원소의 삭제가 O(1)
3. 제일 앞/뒤 원소 확인이 O(1)
4. 제일 앞/뒤가 아닌 원소들의 확인/변경이 원칙적으로 불가능
but stl deque 에서는 index로 접근할 수 있음

구현

const int mx=10000005;
int dat[2*mx+1];
int head=mx, tail=mx; -> 양쪽 끝에서 삽입, 삭제가 가능하기 때문에 중간 지점에서 시작

앞쪽과 뒤쪽의 추가/제거가 모두 필요하면 stl deque가 효율적이지만
굳이 앞쪽에서의 추가와 제거가 필요하지 않으면 stl vector를 사용하는게 효율적
